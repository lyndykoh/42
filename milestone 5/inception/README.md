## Description
**Inception** is a System Administration project designed to broaden knowledge of system virtualization. The project involves building a complete web infrastructure using **Docker Compose**. It provisions a secure NGINX web server (via TLS), a WordPress application, and a MariaDB database, each running in separate, isolated containers.

## Project description
This project utilizes **Docker** to ensure a reproducible and portable infrastructure. Below are the key design choices and comparisons made during development:

* **Virtual Machines vs Docker**: 
    A VM virtualizes hardware to run a full Guest OS, making it heavy and slow. Docker virtualizes the OS kernel, sharing it with the host, which makes containers lightweight and fast while maintaining process isolation.
* **Secrets vs Environment Variables**: 
    Environment variables are suitable for non-sensitive config. However, sensitive data (passwords/keys) should ideally be handled via Docker Secrets (files) to prevent them from appearing in process logs or `docker inspect` outputs.
* **Docker Network vs Host Network**: 
    We use a dedicated Bridge Network. The Host network would remove isolation between the host and containers. A Bridge network allows us to control exactly which ports are exposed (only 443 for NGINX) while allowing internal communication between MariaDB and WordPress.
* **Docker Volumes vs Bind Mounts**: 
    Volumes are the preferred mechanism for persisting data generated by and used by Docker containers. While Bind Mounts depend on the directory structure of the host, Volumes are managed by Docker and are more performant on non-Linux hosts.
